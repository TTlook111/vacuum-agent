# 🤖 vacuum-agent (扫地机器人智能客服助理)

本项目是一个面向垂直领域（扫地/扫拖一体机器人）的智能客服大模型 Agent 实践。基于 LangChain 框架与大模型（默认对接通义千问）构建，采用 ReAct (Reasoning and Acting) 范式，赋予大模型自主思考与工具调用的能力。

## ✨ 核心特性 (Features)

- **🧠 ReAct 核心引擎**: Agent 具备「思考-调用工具-观察-总结」的完整闭环能力，能够处理复杂的多步查询任务。
- **📚 垂直领域 RAG (检索增强生成)**: 
  - 集成 Chroma 向量数据库，并使用 DashScope 文本嵌入模型 (`text-embedding-v4`)。
  - 内置《扫地机器人100问》、《故障排除》、《维护保养》等专业知识库，精准解答常规常识无法覆盖的专业问题。
- **🛠️ 丰富的 Tools 集成**: 
  - **基础信息获取**: 天气查询、用户地理位置定位。
  - **业务数据联动**: 动态提取用户 ID、当前月份，支持检索外部 CSV 用户设备使用记录。
  - **报告生成机制**: 内置 `fill_context_for_report` 等上下文注入工具，支持为用户生成个性化的「设备使用情况与保养建议」报告。
- **🛡️ 健壮性设计**: 向量库入库阶段支持网络抖动重试（指数退避策略）、大文件分批处理及 MD5 文件级去重。

## 🚀 架构演进路线 (Roadmap)

本项目正处于持续重构与迭代中。当前的工具 (Tools) 主要通过本地 Python 函数配合 `@tool` 装饰器实现（包含部分 Mock 数据），主要用于验证 Agent 核心调度逻辑。

**下一阶段核心目标：接入 MCP (Model Context Protocol)**
- [ ] **解耦业务工具**: 将天气查询、用户设备数据获取等业务逻辑剥离为独立的 MCP Server。
- [ ] **标准化调用**: 将当前 Agent 升级为标准的 MCP Client，实现外部能力的动态发现与即插即用。
- [ ] **RAG 服务化**: 探索将 Chroma 检索逻辑封装为独立知识库 MCP Server 的可能性。

## 📁 核心目录结构

```text
├── agent/            # Agent 核心逻辑 (ReAct 循环、中间件、Tools 定义)
├── config/           # 配置文件 (模型配置、向量库参数、Prompt 路径)
├── data/             # 知识库源文件 (PDF, TXT) 与外部数据源 (CSV)
├── model/            # 大模型工厂模式封装 (LLM & Embeddings)
├── prompts/          # 提示词工程 (主系统 Prompt、RAG 总结、报告生成)
├── rag/              # 向量数据库服务 (文档加载、分片、向量化、检索)
└── utils/            # 通用工具类 (日志、文件处理、路径管理)
```
